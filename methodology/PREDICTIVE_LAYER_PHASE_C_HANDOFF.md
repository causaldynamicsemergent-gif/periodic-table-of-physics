### Shipped milestones

- **Step 0 — scope memo.** `PREDICTIVE_LAYER_PHASE_C_SCOPE_MEMO.md` committed to `/methodology/`. Settles six decisions: (1) `quantitative_scale` is a permitted optional field on five distinct surfaces (frontier nodes, cells, predictions, `bears-on` edges, `if_real_implies` implications) with uniform structure; (2) `quantitative_scale` is a structured object with `kind` discriminator (eight values: `energy_scale`, `mass`, `length`, `time`, `ratio`, `coupling`, `dimensionless`, `sigma_deviation`), `value`, `units` (required for dimensional kinds, forbidden for dimensionless), optional symmetric or asymmetric `uncertainty`, optional `log10`, required `citations` (`minItems: 1`); (3) `resolves` is a separate edge type (NOT a strengthening of `bears-on:partially-solves`) with source typed to `experimental-program`, target either cell-id or frontier/totality-approach id, `predictions_per_program` as the load-bearing discriminator substructure, `timeline` enum with six values, no migration of existing edges; (4) firewall admissibility test — three-part test (value is a literature value, per-program predictions are program-internal calculations, sensitivities cite the experimental design) binding for every Phase C authoring pass and required verbatim in the v18 spec; (5) `quantitative_scale` is orthogonal to Phase A's `constructive_status` and composes cleanly with Phase B's `if_real_implies` implications (the natural locus for forward-looking quantitative claims); (6) **the experimental-program inventory gap** — the 7 existing experimental-program nodes are all historical SM-particle-discovery collaborations; `resolves` edges have no source-end nodes to attach to until forward-looking experimental-program nodes are authored. Memo introduces Phase C Step 0.5 to close this gap before any `resolves` authoring begins. Scope memo is the orientation document for all subsequent Phase C authoring decisions.

- **Step 0.5 — experimental-program inventory pass.** 12 forward-looking experimental-program nodes added to `data/Map_v34_consolidated.json`. - **Step 4.1 — frontier-level `quantitative_scale` on 13 of 17 carriers.** Data version bumped v41 → v42. Schema field `_meta._schema` updated to `Map_v18_schema.json` (fixing the stale Phase-C-Step-0.5 pointer carried over from v41). 13 carriers received entries; 4 were skipped. Entries (kind, value [log10?], units?): `qg-frontier` (energy_scale, 10¹⁹ GeV); `cc-frontier` (ratio, ~10¹²⁰, Weinberg 1989); `hierarchy-problem` (ratio, ~10¹⁷); `strong-cp-problem` (dimensionless, |θ_QCD| < 10⁻¹⁰ from Abel et al. 2020 PSI nEDM); `flavor-puzzle` (ratio, ~10⁶ — y_t/y_e); `matter-antimatter-asymmetry` (ratio, η_B = 6.1 × 10⁻¹⁰ ± 0.04 from Planck 2018); `dark-matter` (dimensionless, Ω_DM = 0.265 ± 0.007 from Planck 2018); `cosmological-models` (sigma_deviation, ~5σ Hubble tension, Riess et al. 2022); `turbulence` (dimensionless, K41 spectrum exponent −5/3); `koide-formula` (dimensionless, Q = 2/3 to ~5 decimal places); `muon-g-2` (sigma_deviation, 4.2σ from TI-2020 dispersive HVP); `chpt` (energy_scale, Λ_χ ≈ 1 GeV); `bh-thermodynamics` (dimensionless, S_BH = A/(4 G ℏ) entropy coefficient 0.25). Skipped (4): `bh-info-paradox`, `measurement-problem`, `ns-regularity`, `topological-phases-classification` — for each, no single literature-stated scalar exists at the carrier level (structural / interpretive / mathematical frontiers); absence is the firewall-faithful outcome per scope memo §4. Validator (Rules 1–33 against v18 schema): 0 new schema errors, 0 rule errors, 0 warnings (4 legacy `constrains` errors tolerated per `PROJECT_INFRASTRUCTURE.md` §2).

  **Methodological principle recorded in this step (binding for Step 4.3 and downstream sweeps):** three carriers (`turbulence`, `chpt`, `bh-thermodynamics`) were included despite their characteristic numerical commitment being the *established baseline* of the totality-approach rather than the *strict open-question scale* in the narrower reading the maintainer flagged early in the Step 4.1 proposal. The maintainer's call: "*a periodic table of physics is not complete without thermodynamics; what if some of the potential closures or unifications can't be seen because we left it off the map — it will disrupt patterns*." The principle is **include any characteristic numerical commitment when a literature value is citable, regardless of whether the carrier's open question lives in-scope or is dispatched elsewhere; omission risks distorting cross-FC pattern-finding** (the dimensionless 1/4 in S_BH = A/(4 G ℏ), the K41 exponent, the chiral scale all carry latent pattern-visibility across the map that would be invisible if recorded only on FCs where the open question is in-scope). Skips remain firewall-faithful only when no literature value is citable at all (the 4 SKIPs above qualify; bh-thermodynamics did not, which is why it was promoted out of the initial SKIP list). **The principle propagates to Step 4.3** (cell-level `quantitative_scale`): cells where a characteristic numerical commitment is citable in the literature should receive entries even when their FC's open question lives elsewhere, on the same pattern-visibility grounds. Step 4.4 (`resolves` edges) and Step 4.5 (`if_real_implies` quantitative scales) are not affected by this principle (those mechanisms have stricter scope-internal selection criteria).

### What's queued (live)

- **Step 4.2 — `quantitative_scale` on predictions.** `quantitative_scale` populated on FC-level `predictive_yield[]` entries and cell-level `predictions[]` entries where the prediction has citable numerical content. 223 predictions in v42 (209 FC-scoped + 14 cell-scoped); many are status-only (`confirmed` / `falsified` / `partial` / `aspired` / `unconfirmed-tension`) without an associated number, and will not receive a `quantitative_scale` entry. The expected populated subset is on the order of 100–150 predictions where the cited result names a specific value (e.g., the top-quark mass with measurement uncertainty, neutron-EDM upper bound, Hubble parameter, scaling exponents in real critical systems). Each entry applies the scope memo §4 admissibility test; absence is firewall-faithful where the literature is status-only without a quantitative claim. Authoring split by FC subfield is the natural granularity (one PR per FC or per small batch of FCs). The Step 4.1 methodological principle (include any characteristic numerical commitment when literature-citable) does not change the §4 admissibility bar here — predictions are about specific cited values, not about characteristic commitments, so the principle is moot. Diff shape per PR: small-to-medium; aggregate diff over the full sweep is large.

- **Step 4.3 — `quantitative_scale` on cells.** Cell-level `quantitative_scale` on the subset of cells where a characteristic numerical commitment is the load-bearing claim (Step 4.1's methodological principle applies here directly). Estimated subset: 80–150 cells out of 484 — masses on `sm-rep-content`, `su5-gut-rep-content`, `neutrino-sector-phenomenology`, `hadronic-states`; coupling strengths and characteristic energies on `electromagnetic-phenomena-across-energy-regimes`, `qcd-phase-diagram`, `cft-bootstrap-exclusion-regions`; critical exponents and dimensionless numbers on `real-critical-systems`, `fractional-quantum-Hall-states`, `topological-insulator-and-superconductor-materials`; cosmological parameter values on `cosmological-observations`. Apply the §4 admissibility test entry-by-entry. The Step 4.1 principle: include the commitment when literature names it, regardless of whether the cell's host FC has its open question in-scope or dispatched elsewhere. PR granularity: one FC at a time or small batches of related FCs.

- **Step 4.4 — `resolves` edges.** From each Step-0.5-authored experimental-program node to its target cells and frontiers, with `predictions_per_program` populated where candidate-foundational programs make discriminating predictions. The 12 Step-0.5 nodes are the source-end inventory; the target-end inventory is the carriers and cells those experiments aim at. Apply the §4 three-part test on every edge: literature value on sensitivity, per-program predictions citing program-internal calculations, citations on the experimental design. Expected edge count: 30–60 across the 12 source nodes. This is the Phase C step most likely to surface authoring decisions that need maintainer review (which programs count as "candidate-foundational" for `predictions_per_program`, when an exclusion-only `resolves` edge is admissible vs not, how to handle experimental upgrades / phase transitions in `timeline`). Author one source node at a time, escalating decisions as they arise rather than batching them.

- **Step 4.5 — `quantitative_scale` on `if_real_implies` implications.** Subset of the 24 Phase B implications across 14 carriers where the implied structural consequence is quantitative (e.g., "if SUSY at TeV is real, superpartner masses ≲ 10 TeV" gets a `quantitative_scale` on the implication entry, citing the SUSY-internal paper that names the mass bound). Expected populated subset: roughly half the 24, since many implications are structural ("a new FC is forced") rather than quantitative. The §4 firewall test applies — the implied value must be sourced from the literature that proposed the resolution, not estimated by the cartographer. Smallest of the five authoring sweeps; one PR likely.

## Working norms

Carry forward from `PREDICTIVE_LAYER_PHASE_B_HANDOFF.md` and `TRACKS_AFTER_PHASE_A.md`:

- **Non-developer maintenance.** All deployment via GitHub web UI and Cloudflare web editor. No git CLI, no local dev environment, no `npm` or `wrangler`.
- **Project files are for stable reference material.** This handoff and the scope memo go in project knowledge. Don't attach large schemas, datasets, or worker source to project knowledge — fetch via `web_fetch` instead.
- **CI gates PRs to data + schema.** Methodology docs (this handoff, the scope memo, the v18 spec) commit directly to `main` without CI gating. Data and schema PRs must pass `scripts/validate.py` (Rules 1–33 since Step 3).
- **The firewall is binding, and Phase C's admissibility test is binding within it.** `META_v21_1_methodology_firewall.md` is the parent; the scope memo §4 / v18 spec §4 three-part test is Phase C's operationalization. The v18 spec includes the test verbatim; every authoring step records the §2.5 self-check in the PR description.
- **Step 4.1's methodological principle is also binding for Step 4.3.** Include any characteristic numerical commitment when a literature value is citable, regardless of whether the carrier's open question lives in-scope or is dispatched elsewhere. Omission risks distorting cross-FC pattern-visibility. The principle does not relax the firewall — citations are still required per the §4 test; the principle is about which surfaces qualify for inclusion when the literature is citable, not about what counts as citable.
- **The MCP source is in the repo.** `/mcp/worker.js`, `/mcp/worker_skeleton.js`, `/mcp/build_worker.py`, and `/mcp/README.md` are version-controlled. The Phase C Step N+1 rebuild edits the skeleton, not the bundled artifact. Note (per Where-things-stand drift): the deployed worker on Cloudflare appears to lag the repo source; the Step N+1 rebuild should diff-check repo source against deployed worker before declaring the rebuild done.
- **Existing content is load-bearing.** Cell content, axis labels, citation text, edge descriptions, status fields, frontier and totality-approach node fields (`candidate_targeting`, `equations`, `loose_ends`, `description`, `empirical_status`), Phase B's `if_real_implies` entries, and Phase C's `quantitative_scale` entries are not paraphrased in commits or code. Phase C adds the `quantitative_scale` field and `resolves` edges; it does not rewrite existing fields.
- **Use physics vocabulary.** User-stated principle. Don't coin presentation terms when the data already names a thing.
