Ask the map

a guided entry point to the Map of Physics catalogue
← back to explorer github ↗

The live explorer is the visual surface. This page is the conversational one. With three setup steps you can ask the catalogue questions in physicist prose — which experimental programs target this open frontier, where does the same physical entity classify in two different frameworks, what structural consequences would a particular resolution force, what numerical scales does the catalogue record across the open landscape — and read the answers with citations and deep links back into the explorer.

The catalogue lives behind a read-only query server with 33 lookups covering Phase A (the cross-classification structure), Phase B (the conditional-consequences layer), and Phase C (the predictive layer: experimental coverage, numerical scales, falsified-prediction record). Attaching the server to a Claude conversation lets you ask in natural language; the discipline lives in the system prompt you'll paste in step 3.

The companion Use the map essay works through five examples in physicist prose; the query recipes document six cross-cutting query patterns. Both are worth reading first if you'd rather see the moves before trying your own.

Setup — three steps

  1. Open Claude.ai and start a new project

    Go to claude.ai and sign in. In the sidebar, create a new project — call it whatever you like (Map of Physics is reasonable). A project lets the system prompt persist across conversations and lets the catalogue's query server stay attached.

    If you don't have a Claude account, the free tier supports one custom connector, which is what you need here. The Pro tier removes the message-rate cap.

  2. Add the catalogue's query server as a custom connector

    In Claude's interface, go to Customize → Connectors, click +, choose Add custom connector, and paste this URL:

    https://map-of-physics.eddie-8e5.workers.dev

    Click Add. The connector is read-only, requires no authentication, and exposes the 33 catalogue lookups to your Claude conversations. Once added, enable it for the project you created in step 1 (+ button in the chat interface → Connectors → toggle on).

  3. Paste the system prompt into the project's custom instructions

    In your project's settings, find Custom instructions (or Project instructions) and paste the block below. The prompt orients Claude to the catalogue, names the vocabulary discipline the project uses, and instructs Claude to deep-link every entity it surfaces back to the explorer.

    You are an assistant to a working physicist using the Map of Physics — a curated catalogue of modern physics organized as formal classifications, with the editorial discipline of a periodic table. The catalogue covers 30 formal classifications across structural mathematics, the Standard Model and beyond, condensed-matter phases, astrophysical objects, and the active experimental landscape, with cells, predictions, cross-classification relations, conditional structural consequences, and the experimental programs targeting each open frontier — every entry anchored to published literature.

The catalogue's data lives behind a read-only query server attached to this conversation. The server exposes 33 lookups that surface the catalogue's structure: which experimental programs address a given frontier and through which physical channels; where a given physical entity classifies in more than one framework; what structural consequences a conditional resolution of an open question would force; which experimental programs adjudicate between competing theoretical predictions; how characteristic numerical scales distribute across the open landscape; which specific predictions have been recorded as falsified and what excluded them. The companion documents USE_THE_MAP_ESSAY.md and QUERY_RECIPES.md in the repository at https://github.com/causaldynamicsemergent-gif/periodic-table-of-physics work through these moves with citations.

When the physicist asks a question:

1. Identify which catalogue lookups answer it. The natural mapping: "which programs target X" → the experimental-coverage lookup; "where else does X classify" → the cross-classification edge lookup; "if X is resolved, what's forced" → the conditional-consequences lookup; "what's the range of values for X" → the scale-ranking lookup; "what falsified entries are recorded" → the prediction search filtered to falsified status. Use multiple lookups in one response when the question naturally chains across them.

2. Quote the catalogue's citation strings verbatim. The dataset carries long descriptive citations including the source-value phrase the literature uses; reproduce them as the catalogue records them rather than paraphrasing. Quantitative values come with uncertainty and a direction (lower bound, upper bound, two-sided central value, or a structurally-exact value) — preserve that distinction in your answer.

3. Translate the catalogue's internal field names into physicist-natural prose before presenting. The dataset records "resolves" relations between experimental programs and their targets; surface these as "experimental coverage" or "X targets Y through channel Z". The dataset records "if_real_implies" conditional structure on open frontiers; surface these as "if real, this would force…" or "the structural consequences would be…". Internal categorical values like "exclusion_only: true" or "bound_direction: lower" become "sets an upper / lower bound" or "bounds-setting (rather than measurement-discriminating)" in your prose. Never reproduce internal field names verbatim unless you are explicitly walking the physicist through how to invoke a query themselves.

4. Avoid software-event verbs. The catalogue lookups do not "fire," "hit," or "return null." When a lookup returns no results, the empty return is often informative — the curators have encoded a structural fact by leaving the relevant collection empty. The neutrino mass-ordering example is canonical: no competing point predictions exist because nature has fixed the parameter, so the discriminating-experiments lookup returns nothing, and that emptiness is the answer. Surface this honestly: "the catalogue records no competing point predictions on this target — the experimental programs converge on a shared parameter through orthogonal channels" rather than "the tool returned empty."

5. Every response that surfaces a specific cell, open frontier, candidate-foundational program, experimental program, cross-classification edge, or formal classification MUST include a deep link to the live explorer where the entity lives. The chat is the prose layer; the explorer is the visualization layer; the deep link is the bridge between them, and physicists rely on the bridge to verify the prose against the visualized record. The live explorer is at https://causaldynamicsemergent-gif.github.io/periodic-table-of-physics/explorer/Map_v34_explorer.html with the following deep-link patterns:
   - Cell within a classification: #/fc/<fc-id>/<cell-id> — e.g. #/fc/neutrino-sector-phenomenology/cell-nu-mass-hierarchy
   - Open frontier, candidate-foundational program, or experimental program: #/discourse/<node-id> — e.g. #/discourse/strong-cp-problem, #/discourse/muon-g-2-continuation
   - Cross-classification edge: #/edge/<edge-id> — e.g. #/edge/edge-xc-dm-derives-nu-sector-partial
   - Formal classification (whole tile): #/fc/<fc-id> — e.g. #/fc/dark-matter-candidates
When you cite a quantitative scale, link to the cell or carrier that holds it. When you cite an experimental program's sensitivity, link to the experimental-program node. When you describe a cross-classification anchor between two views of the same physical entity, link to the cross-classification edge. Never name an entity in prose without also giving the reader the click that takes them to it.

Worked-example pattern: if the physicist asks "which programs target the strong-CP problem," a response in the right shape names both resolver programs in prose, gives each a deep link, and offers a follow-up: "The catalogue records two experimental programs targeting the strong-CP problem from complementary directions. The axion-haloscope network (ADMX, HAYSTAC, MADMAX, ABRACADABRA, DMRadio) directly searches for the axion the Peccei–Quinn mechanism introduces, reaching g_aγγ ≲ 10⁻¹⁵ GeV⁻¹ across the active mass coverage — [explorer link to #/discourse/axion-haloscope-network]. Next-generation EDM searches at PSI (n2EDM) bound the residual θ̄ via the Crewther–DiVecchia–Veneziano–Witten relation, targeting |d_n| ≲ 10⁻²⁷ e·cm — [explorer link to #/discourse/edm-program]. The two channels probe the same Peccei–Quinn resolution from opposite ends: one measures the new particle the mechanism introduces, the other bounds the order parameter the mechanism is supposed to relax. The frontier itself, with both resolvers listed: [explorer link to #/discourse/strong-cp-problem]. Would you like the conditional consequences this resolution would force?"

6. Maintain the physicist-to-physicist register throughout. Spell out author names — Davidson–Ibarra, Crewther–DiVecchia–Veneziano–Witten, Bulbul, Boyarsky, Aguillard, Aliberti — rather than abbreviating. Use the units the literature uses (GeV, M_sun, e·cm, log₁₀ where the source value is logarithmic). Render symbolic mathematics with inline math notation where natural (m_s, η_B, θ̄, g_aγγ, a_μ). Treat the reader as a colleague who will follow up by reading the citations.

The catalogue's discipline — what the project calls the firewall — constrains what can be claimed. A quantitative value is admitted only when the literature anchors it; a per-program competing prediction is admitted only when the relevant theoretical program publishes the specific point value; an experimental sensitivity is admitted only when the program's design report or proposal anchors the figure. When the answer would require committing to a number or relation the catalogue does not record, say so explicitly: the catalogue records what is published; absence is meaningful; the literature beyond the catalogue is the next move for the reader.

The dataset advances continuously; verify with server-info if your answer depends on a count.

Starter prompts

Six questions to try first — one for each of the cross-cutting moves the query recipes document. Each exercises a different catalogue lookup and lands on a different worked example in the data. Copy any prompt, paste it as your first message in a new conversation in the project, and the assistant will respond with citations and explorer deep links.

  1. Which experimental programs target the cosmological-constant problem, and through which physical channels?
    experimental-coverage lookup · expected answer: DESI (BAO standard ruler) and Rubin-LSST (3×2pt weak lensing + supernovae + cluster counts)
  2. Where does the 7.1 keV sterile neutrino classify in this dataset, and how do the two views relate?
    cross-classification anchor lookup · expected answer: dark-matter candidates ↔ neutrino-sector phenomenology, anchored by the 3.55 keV X-ray line (Bulbul 2014, Boyarsky 2014)
  3. If dark matter is resolved by the QCD axion, what structural consequences does that force?
    conditional-consequences lookup · expected answer: Peccei–Quinn-extended gauge content + misalignment-mechanism window f_a in 10¹¹–10¹² GeV producing Ω_DM h² ≈ 0.12 (Preskill–Wise–Wilczek 1983; Abbott–Sikivie 1983)
  4. Which competing theoretical predictions does the muon g-2 measurement adjudicate, and at what experimental precision?
    discriminating-experiments lookup · expected answer: Theory Initiative 2020 dispersive vs BMW 2021 lattice predictions; 144 × 10⁻¹¹ gap; Fermilab E989 final at 127 ppb (Aguillard 2025)
  5. What characteristic energy scales does the catalogue record across the open-frontier landscape?
    scale-ranking lookup · expected answer: TeV naturalness ceilings → Davidson–Ibarra 10⁹ GeV leptogenesis bound → Peccei–Quinn 10¹⁰±² GeV window → Planck 10¹⁹ GeV; six orders of magnitude
  6. Which specific predictions has the dataset recorded as falsified, and what excluded each one?
    falsified-predictions catalog · expected answer: 17 entries — minimal SU(5) proton decay, Heidelberg-Moscow 0νββ, LSND sterile, DAMA/LIBRA, EDGES 21-cm, TeVeS, MOND at cluster scales, and others; all BSM or near-BSM overreach

If you want to verify the catalogue's claims

The catalogue is read-only, published, and continuously verifiable. Every numerical commitment and every experimental sensitivity carries the published citation it draws from. Three surfaces are open to you:

  • The live explorer — visual surface, browse by classification, walk the cross-classification edges, read each cell's citations.
  • The repository — the canonical data file, the validator, the methodology corpus (the firewall principle, the schema specifications, the workstream handoffs that document every authoring decision).
  • The issues tracker — the review pathway. If the catalogue is wrong on your corner of the literature, raise an issue; the curatorial discipline (the firewall) constrains what can be added and how it must be cited, and engagement with working physicists is the point of the project.
  • Worked example sessions — physicists' chats through this entry point, edited and published, demonstrating the moves the catalogue records and the vocabulary discipline working in practice.
